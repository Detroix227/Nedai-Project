# NedAI: User Academic Profile Setup & Dashboard Documentation

## Overview
The User Academic Profile is a foundational component of the NedAI ecosystem. Unlike generic AI chatbots, NedAI requires an understanding of the user's specific educational context to provide curriculum-aligned and pedagogically safe tutoring. The Academic Profile Setup and the subsequent Dashboard allow learners to define their academic parameters, which directly influence how the Retrieval-Augmented Generation (RAG) pipeline and the Hybrid Large Language Models (LLMs) interact with them.

## 1. Academic Profile Setup (Onboarding)
Upon successful registration and authentication, users are immediately directed to the Academic Profile Setup screen. This onboarding step is critical for grounding the AI.

### Key Data Collected:
* **Educational Level:** Users select their current standing within the Nigerian education continuum (e.g., ECCDE, Primary, Junior Secondary School, Senior Secondary School, or Tertiary). This selection dictates which segment of the Knowledge Vault is prioritized during context retrieval.
* **Institution:** The specific school or university the student attends, providing additional context for the AI.
* **Learning Goals/Preferences:** Primary academic objectives (e.g., "WAEC Preparation", "Undergraduate Research", "Remedial Math"). This helps the system tailor its initial conversational style and recommendations.

## 2. User Dashboard
The User Dashboard serves as the central hub for the learner's interaction with NedAI. It presents a dynamic overview of the user's academic profile and ongoing system interactions.

### Dashboard Components:
* **Profile Summary:** Displays the user's current educational level, institution, and explicitly stated goals.
* **Dynamic Learning Profile (UserLearningService Insights):** 
    * The dashboard visualizes background data generated by the backend `UserLearningService`.
    * **Inferred Preferred Tone:** Displays the system's active behavioral modality (e.g., *Reflective*, *Analytical*, *Guide*) based on its analysis of recent chat interactions.
    * **Subject Proficiency/Weak Subjects:** Highlights subjects where the user frequently requests help or expresses difficulty, aiding the student in focusing their study efforts.
* **Knowledge Vault Quick Access:** Provides a snapshot of the user's uploaded curriculum materials (PDFs/DOCX) and their current ingestion/vectorization status.

## 3. Underlying Technical Architecture
The profile setup and dashboard functionality are supported by the following stack components:

* **Database Schema (PostgreSQL):** 
    * `users` table: Stores static user inputs such as `academicLevel` and `learningPreferences`.
    * `user_learning_profiles` table: Continuously updated by the backend, storing dynamically inferred traits like `preferredTone` and `weakSubjects`.
* **State Management:** The frontend (built with React/Vite) utilizes **Zustand** to maintain the user's profile state. This ensures the dashboard updates seamlessly when the `UserLearningService` flags a change in the user's learning profile, without requiring page reloads.
* **Privacy and Security:** All user profile data is managed securely, ensuring compliance with the Nigerian Data Protection Act (NDPA).
